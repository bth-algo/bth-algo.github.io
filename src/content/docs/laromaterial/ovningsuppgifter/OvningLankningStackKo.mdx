1. 
    <details>
    <summary>Klicka här för att visa svaret</summary>

    ```
    if current is none
        return 0
    walker = current.next
    counter = 1
    while walker != current
        counter = counter + 1
        walker = walker.next
    return counter
    ```
    </details>

1. 
    <details>
    <summary>Klicka här för att visa svaret</summary>

    ```
    if current.next is none
        current = none
    else
        temp = current
        current = current.next
        temp.prev.next = current
        current.prev = temp.prev
    ```
    </details> 

1. 
    <details>
    <summary>Klicka här för att visa svaret</summary>

    ```
    if first is none or toInsert.data < first.data
        toInsert.next = first
        first.prev = toInsert
        first = toInsert
    else
        walker = first
        while walker.next is not none and toInsert.data > walker.next.data
            walker = walker.next
        toInsert.next = walker.next
        toInsert.prev = walker
        walker.next.prev = toInsert
        walker.next = toInsert
    ```
    </details>


1. 
    <details>
    <summary>Klicka här för att visa svaret</summary>
    En Stack som är implementerad med en länkad lista är lämpligare eftersom den inte har en fast kapacitet utan den kan växa dynamiskt i takt med att element läggs till. En Stack som är implementerad med en array har en fast kapacitet och när denna kapacitet överskrids måste en ny större array skapas och innehållet kopieras över, på samma sätt om det är få element i arrayen i förhållande till kapaciteten så kan en mindre array skapas och innehållet kopieras över, vilket kan vara ineffektivt.
    </details>

1. 
    <details>
    <summary>Klicka här för att visa svaret</summary>
    En Kö som är implementerad med en array är lämpligare eftersom den har en fast kapacitet och när denna kapacitet inte överskrids så är det mer minnes- och tidsmässigt effektivt än att använda en länkad lista där varje element kräver extra minne för att lagra pekare. 
    </details>
