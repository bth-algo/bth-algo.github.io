## Förutsättning

Du har uppfyllt kraven i uppgiften och har gjort en vanlig inlämning enligt [Resultat och redovisning](./resultat-och-redovisning).


## Ordlista

- `PR`: Pull request
- `Tests passed(TP)`: Får din PR när de automatiska testerna gått igenom. Nu ska du bara vänta på att en lärare kollar på din inlämning.
- `Tests failed`: Får din PR när något gått fel i de automatiska testerna. Ladda ner testerna igen, fixa felet lokalt och pusha till GitHub.
- `Grade: G`: Helt klar med en uppgift.
- `Grade: Ux`: En lärare har bedömt din inlämning men tycker att något behöver förbättras i din inlämning. Följ instruktionerna, fixa problemet lokalt och pusha din lösning till GitHub. När du taggar så körs testerna på GitHub.
- `Grade: U`: En lärare har bedömt din inlämning men tycker inte att det är tillräckligt bra. Du behöver göra en ny inlämning till nästa rättningstillfälle.
- `Run tests`: Du ska sätta labeln "Run tests"  för att köra igång flödet som kör testerna och sätter `Tests Passed (TP)` eller `Tests failed`.
- `Integration failed`: Något i koden som kör de automatiserade testerna gick fel. Kontakta en lärare.


## Mitt eget arbete

Det du lämnar in är ditt eget arbete. Om du har tagit hjälp av extern kod, eller om du har kopierat någon mindre del av koden, så är det tydligt angivet i kommentarerna i koden eller i din redovisningstext.

I samband med att du gör din PR så intygar du att arbetet är ditt eget och uppfyller de regler som finns för "[Fusk och medveten vilseledning](/studieguide/fusk/)".

### Testerna gick fel

Om något gick fel med testerna får du en kommentar om vad som gick fel samt din PR får labeln `Tests failed` och PR:n stängs. Eftersom du kör testerna när du taggar så bör detta inte inträffa.

Vanliga fel:

- du har inte de nyaste testerna
- du inte inloggad med SSO (logga ut och sen in igen och klicka på SSO-knappen på rätt organisation)

Nu behöver du kolla vad som gick fel, fixa felet och ladda upp koden igen. Enklast gör du det genom att:  

1. Ladda ner testerna på nytt och kör dem lokalt
1. Rätta felet och testkör lokalt.
1. Pusha din rättning till GitHub och tagga den
1. Sätt labeln `Run tests` på din öppna PR för att köra igång flödet som kör testerna och sätter `Tests Passed (TP)` eller `Tests failed`.


:::tip
Vi rättar inte inlämningar (PRs) med labeln `Tests failed`.
:::


## PR kmom06

### Skapa branchen `bth/submit/kmom06`

Branchen `bth/submit/kmom06` saknas i ditt repo och den skapar du enklast i terminalen. Det första kommandot kollar vilka brancher det finns och markerar den aktuella branchen med en `*`. Det andra kommandot gör din branch aktuell och håller historiken ren med flaggan `rebase`.

```bash title="Verifiera att du står i main-branchen (* framför main) och att den är aktuell"
git branch
git pull --rebase
```

Det första kommandot skapar branchen och det andra kommandot pushar upp det till GitHub. Om du kör `git branch` nu så ser du att `*` är framför `bth/submit/kmom06`. Med det tredje kommandot hoppar du tillbaka till main där du ska jobba.

```bash title="Skapa en ny branch, pusha den och hoppa tillbaka till main"
git switch -c bth/submit/kmom06
git push origin bth/submit/kmom06
git switch main
```

Om du kör `git branch` nu så ser du att `*` är framför main-branchen.

### Skapa en PR för kmom06

Gör på samma sätt som i [Pull Request (PR)](./#pull-request-pr).

## Feedback och bedömning

När läraren rättar din PR så får du återkoppling i kommentarer till din PR.

### Godkänd - Grade: G

Om läraren tycker att allt ser bra ut får du en kommentar, label `Grade: G` på din PR och `G` i Canvas. Läraren mergar din PR och stänger den när den är godkänd. Du kommer också se vilken lärare som har varit "reviewer". Du kan jobba vidare med nästa kmom.


### Komplettering - Grade: Ux

Om läraren tycker att något är fel som du behöver fixa får du label `Grade: Ux` på din PR och `Ux` på Canvas. Det kommer finnas kommentarer på specifika rader. Om kommentaren innehåller en `Warning` som på bilden nedanför, då måste du åtgärda det i din kod. Om kommentaren inte innehåller `Warning` behöver du inte åtgärda det som står där.

import pr_warning from '@assets/pr/pr_comment_warning.png';

<Figure 
    src={pr_warning}
    caption="Kommentar som ska åtgärdas."

När du har åtgärdat felen gör följande:
 
1. Rätta felet och testkör lokalt.
1. Pusha din rättning till GitHub och tagga den. Om din fix funkade ska du nu få labeln `Tests passed(TP)`.
1. Sätt labeln `Run tests` på din öppna PR för att köra igång flödet som kör testerna och sätter `Tests Passed (TP)` eller `Tests failed`.

:::note
Din komplettering ska lämnas in inom 5 arbetsdagar!
:::


### Inte godkänd - Grade: U

Om din inlämning inte är tillräckligt bra så får den labeln `Grade: U` på din PR och `U` i Canvas. Det kommer finnas kommentarer med motivering och läraren kommer att stänga din PR.

Vid större fel, ges inte komplettering. Då kan du få `U` direkt.

Din inlämning kommer inte rättas mer. Din PR stängs. Du behöver göra en ny PR till nästa rättningstillfälle. I [studieplanen](/intro/studieplan/) hittar du när nästa rättningstillfälle är.
