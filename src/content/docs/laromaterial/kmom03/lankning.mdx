Genom att skapa objekt som kan *peka ut* andra objekt av samma sort kan vi bland annat skapa kedjor av objekt - så kallade länkade listor. Objekten kallas ofta *listelement* eller *noder*.  

<br></br>
<Figure
    src={enkelLankning}
    width="500px"
/>

På bilden ser vi en **enkellänkad lista**. Det framgår att det finns en variabel, i detta fall namngiven *first*, som pekar på den första noden som innehåller datan/informationen A. Den första noden pekar i sin tur på ett annan nod som innehåller B o.s.v. Det sista noden, som innehåller G, pekar inte vidare på något annan nod.

Det är möjligt att nå den första noden via variabeln *first*. För att nå den andra noden behöver vi börja från *first* och gå via den första noden genom att använda *pekaren* i denna. Pekaren i första noden pekar på den andra noden. Listan vi ser är riktad (pekarna går åt ett håll) och kallas för en **enkellänkad lista**.

Vid länkning behöver vi definiera en klass (eller motsvarande) som ska användas för att representera objekt:  

- kan hålla data/info och 
- kan peka ut ett (eller flera) objekt som är av samma klasstyp

## Enkellänkad lista med heltal

I detta fall kommer vi låta medlemsvariablerna vara publika - dels för att direkt (utan metodanrop) kunna ändra på den referens som motsvarar *pekaren* och dels för att vi senare kan se att det är ok om vi kapslar in klassen och döljer den inom den klass som ska skapa länkade konstruktioner. Den yttre klassen "äger" då den inre klassen och har förtroendet att accessa medlemsvariablerna i denna.  
Låt oss börja med att definiera en klass som kan hålla ett heltal och länka den till ett annat objekt av samma sort.

```csharp
public class Node
{
    public int Data; // värdet som ska lagras
    public Node? Next; // referensen till nästa Node-objekt
    public Node(int data, Node? next = null)
    {
        this.Data = data;
        this.Next = next;
    }
}
```
Nu kan vi skapa objekt.

```csharp
Node first = new Node(10); 
Node second = new Node(20); 
Node third = new Node(30); 
```

Ändra `Next` i det objekt `first` pekar på till referensen som objektet med värdet 20 pekar på. Denna referens finns i variabeln `second`.
Ändra `Next` i det objekt `second` pekar på till referensen som objektet med värdet 30 pekar på. Denna referens finns i variabeln `third`.

I C# blir detta:

```csharp
first.Next = second;
second.Next = third;
second = null; // bara för att tydligt släppa referensen till objektet
third = null; 
```

Vi observerar återigen att kedjan löper på ett håll. Det går till exempel att komma från noden med värdet 10 till noden med värdet 20 men inte tvärtom.

För att skriva ut värdet som finns i Data i det andra noden behöver vi börja i `first`, använda punkt-notation för att accessa `Next` i objektet som `first` pekar på, dvs  `first.Next`. Här finns referensen till nästa objekt så vi använder återigen punkt-notation, för att accessa Data, dvs `first.Next.Data`. Det blir alltså

```csharp
    Console.Write(first.Next.Data);
```

## Enkellänkad lista för olika datatyper

Nu vill vi ändra så att Node-objekten kan lagra olika datatyper. Vi gör klassen generisk. I detta fall väljer vi att använd namnet E (för Element) på den obestämda datatypen.

```csharp
public class Node<E>
{
    public E Data;
    public Node<E>? Next;
    public Node(E data, Node<E>? next = null)
    {
        this.Data = data;
        this.Next = next;
    }
}
```

För att skapa objekt för att hantera strängar skriver vi.

```csharp
Node<string> elemOne = new Node<string>("A");
Node<string> elemTwo = new Node<string>("B");
```

För att länka ihop dessa så att ordningen blir "A" följt av "B" skriver vi.

```csharp
elemOne.Next = elemTwo;
```

## Dubbellänkade listor

Vi kan skapa länkade listor där noderna *pekar på* noden före (previus) och noden efter (next). Det ger oss möjlighet att vandra i listan i båda riktningarna.

<br></br>
<Figure

    width="500px"
/>

Klassen behöver ytterligare en medlemsvariabel som kan hålla referensen till den nod som kommer före.

```csharp
public class Node<E>
{
    public E Data;
    public Node<E>? Prev;
    public Node<E>? Next;
    public Node(E data, Node<E>? prev = null, Node<E>? next = null)
    {
        this.Data = data;
        this.Prev = prev;
        this.Next = next;
    }
}
```

## Annan länkning

Det behöver inte vara listor som bildas när vi länkar noder. Senare i kursen kommer vi att se att länkning även kan användas för att skapa andra strukturer såsom träd och grafer.


## Sammanfattningsvis

En länkad lista är en följd av noder/element där varje nod "pekar på" en eller flera andra noder.  
I C# skapas en klass för att representera en nod. Klassen görs dessutom generisk för att möjliggöra för olika datatyper i de länkade listorna.  
Det behövs (minst) en separat pekar som pekar det första elementet i en länkad lista. Därefter kan vi följa nodernas pekare för att nå andra noder.  
I cirkulära listor finns ingen börjar eller slut, men det behövs en separat pekare för att ha någon "ingång" till noderna i listan.
