När BusStop-objekt jämförs ska det vara `_name` som jämförelsen baseras på. Anta exempelvis att följande objekt skapas:

- `busStopB.CompareTo(busStopA)` i -1

### Sökalgoritmer

Algoritmerna ska vara generiska men kräva att elementen som sökningen utförs på implementerar `IComparable`-gränssnittet så att de kan jämföras med varandra.

Det innebär att det bl. a ska vara möjligt att utföra sökning i en array av heltal, en array av strängar och en array anpassade objekt av typen `BusStop` som implementerar `IComparable`-gränssnittet (enligt klassdiagrammet ovan).

I en fil namngiven SearchAlgorithms.cs ska du implementera klassen `SearchAlgorithms` med de två sökalgoritmerna `LinearSearch` och `BinarySearch` (se nedan). Klassen ska ingå i samma namespace (`Searching`) som `BusStop`.   

Det finns för båda sökalgoritmerna en *extra parameter* som är en lista (datatypen `List<int>`) vilken ska användas för att registrera de index där element finns som det sökta har jämförts med. Detta för att kunna analysera hur många jämförelser som görs och var i arrayen dessa jämförelser gjorts.

Parameterlistan ska vara anpassad efter följande anrop:
```
// Sökning av targetInt i arrayen nrs med 10 heltal 
int index = SearchAlgorithms.LinearSearch(nrs, 10, targetInt, indexes);
```
```
// Sökning av targetStr i arrayen words med 100 strängar
int index = SearchAlgorithms.BinarySearch(words, 100, targetStr, indexes);
``` 
```
// Sökning av targetBusStop i arrayen busStops med 1000 BusStop-objekt
int index = SearchAlgorithms.LinearSearch(busStops, 1000, targetBusStop, indexes);
```

```csharp
public class SearchAlgorithms
{
    public static int LinearSearch<T>(/* parameter lista */)
    where T : IComparable<T>
    {
        // Implementera linjärsökning här
    }

    public static int BinarySearch<T>(/* parameter lista */)
    where T : IComparable<T>
    {
        // Implementera binärsökning här
    }
}
```
#### Exempel på anrop och förväntat resultat

##### Linjärsökning

Anta att `nrs` är en heltalsarray med innehållet 20, 40, 30, 70, 60, 50 och att indexes är en tom lista (`List<int> indexes = new List<int>()`).  

Efter anropet `int index = SearchAlgorithms.LinearSearch(nrs, 6, 30, indexes)` ska `index` innehålla 2 och listan `indexes` innehålla 0, 1, 2.

Efter anropet `int index = SearchAlgorithms.LinearSearch(nrs, 6, 90, indexes)` ska `index` innehålla -1 och listan `indexes` innehålla 0, 1, 2, 3, 4, 5.

##### Binärsökning

Anta att `orderedNrs` är en heltalsarray med innehållet 22, 33, 44, 55, 66, 77, 88 och indexes är en tom lista.

Efter anropet `int index = SearchAlgorithms.BinarySearch(orderedNrs, 7, 55, indexes)` ska `index` innehålla 3 och listan `indexes` innehålla 3.

Efter anropet `int index = SearchAlgorithms.BinarySearch(orderedNrs, 7, 33, indexes)` ska `index` innehålla 1 och listan `indexes` innehålla 3, 1.


### Säkerställ att algoritmerna fungerar

När du implementerat sökalgoritmerna behöver du göra egna tester för att säkerställa att algoritmerna fungerar såsom avsett. 

Det är viktigt att du inte bara testar att algoritmerna ger rätt resultat utan också att de gör det på ett effektivt sätt, dvs. att de gör rätt antal jämförelser och att jämförelserna sker på rätt element i arrayen. Till detta använder du den extra parametern av typen `List<int>` .

Några tips på vägen:

1. Fungerar algoritmen för olika datatyper (int, string, BusStop)?
1. Ger algoritmen korrekt svar på 
    - en tom array?
    - om det som söks är placerat först?
    - om det som söks är placerat sist?
    - om det som söks finns på ett index som inte är första eller sista?
    - om det finns flera förekomster av det som söks?
    - om det som söks inte finns i arrayen?
1. Gör algoritmen korrekt antal jämförelser och utförs jämförelserna på rätt element?

I filen `orderedWords1200.txt` finns 1200 ord i sorterad ordning som du kan använda för att testa dina sökalgoritmer på lite större mängder med strängar. Filen hittar du i katalogen `example`.

<details>
<summary>Här är ett förslag på metod för att läsa strängar från fil.</summary>  


```csharp
// skapar en array med strängar som läses från fil. Storleken på arrayen bestäms av värdet i nrOf, om vädret är -1 läses alla strängar som finns på filen
string[] ReadWordsFromFile(string filename, int nrOf = -1)
{
    if (nrOf == -1)
    {
        string[] all = File.ReadAllLines(filename);
        return all;
    }

    string[] words = new string[nrOf];

    using (var reader = new StreamReader(filename))
    {
        string? word;
        int index = 0;

        while ((word = reader.ReadLine()) != null && index < nrOf)
        {
            words[index] = word;
            index++;
        }
    }

    return words;
}
```
</details>
För att testa med större mängder heltal kan du dels generera slumpmässiga heltal genom att använda `Random`-klassen och dels sortera dem med hjälp av `Array.Sort`-metoden. 

<details>
<summary>Här är ett förslag på metod för att skapa en array med slumpade tal i sorterad ordning.</summary>  

```csharp
// skapar en array av storlek nrOf och fyller den med slumpade heltal inom [minValue..maxValue] som sorteras
int[] RandomOrderedNumbers(int minValue, int maxValue, int nrOf, int seed = 1234)
{
    Random random = new Random(seed);

    int[] nrs = new int[nrOf];
    for (int i = 0; i<nrOf; i++)
        nrs[i] = random.Next(minValue, maxValue + 1);
    Array.Sort(nrs);
    return nrs;
}
```
</details>
