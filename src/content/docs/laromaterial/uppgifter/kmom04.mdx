I denna uppgift ska du implementera en ADT:n Cirkulär Lista. Denna lista ska därefter användas för att illustrera en "uträknings"-ramsa.

### Förberedelser

I `kmom04` skapar du ett dotnet console-projekt namngivet `CircularList`.

### Implementation av en ADT Cirkulär lista

Klassen ska namnges `CircularList` och implementera det generiska interfacet `ICircularList` vilket du skapar i filen `ICircularList.cs`.

```csharp
public interface ICircularList<T>
{
    public void InsertAfterCurrent(T item);
    public void InsertBeforeCurrent(T item);
    public T GetCurrent();
    public void RemoveCurrentAndMoveForward();
    public void RemoveCurrentAndMoveBackward();
    public bool Contains(T item);
    public int Size();
    public void MoveCurrentForward(int nrOfSteps);
    public void MoveCurrentBackward(int nrOfSteps);
    public void Clear();
}
```

`CircularList` ska använda dubbellänkning som intern datastruktur.
Vid försök att utföra orimliga operationer på listan ska undantag kastas. Listan har alltid en pekare `current` som pekar på den nod i listan som betraktas som aktuell. Förutom denna pekare har listan även en räknare (heltalsvariabel) som håller antalet element som finns i listan.

När listan är tom har `current` värdet null.

Klassen som definierar en nod i listan ska vara en inre klass. Denna ska bl.a ha en pekare som pekar på efterföljaren (Next) och en pekare som pekar på föregångaren (Prev).


#### Beskrivning av listans metoder

*Anm. I texten nedan anges bl.a riktningen framåt vilket avser att följa Next-pekaren och på motsvarande sätt avser riktningen bakåt att följa Prev-pekaren.*


`InsertAfterCurrent`- placerar elementet som efterföljare till det som current pekar på men current ändras inte.
Om listan däremot är tom blir det nya elementet det enda i listan och current ska peka på detta.

`InsertBeforeCurrent`- placerar elementet som föregångare till det som current pekar på men current ändras inte.
Om listan däremot är tom blir det nya elementet det enda i listan och current ska peka på detta.

`GetCurrent` - returnerar aktuellt element.

`RemoveCurrentAndMoveForward`- tar bort det element som current pekar på och ser till att current pekar på det borttagna elementets efterföljare.

`RemoveCurrentAndMoveBackward`- tar bort det element som current pekar på och ser till att current pekar på det borttagna elementets föregångare.

`Contains` - avgör om elementet finns i listan eller ej.

`Size` - returnerar antalet element som finns i listan.

`MoveCurrentForward` - flyttar current framåt det antal steg parametern innehåller.

`MoveCurrentBackward` - flyttar current bakåt det antal steg parametern nrOfSteps innehåller.

`Clear` - tömmer listan.

### Använda ADT Cirkulär lista

Inom datavetenskap och matematik är **Josephus-problemet** (eller Josephus-permutation) ett teoretiskt problem som motsvarar en sorts ”uträkningslek”, exempelvis *elle-belle-bi-nu-är-du-så-sant-som-fri*. Ramsan *elle-belle-...* kan ersättas med att man räknar från 1 tom ett bestämt tal X där den som blir nummer X är den som får lämna. Därefter börjar man på nytt från 1 och nästa person som blir nummer X får lämna. Detta fortsätter tills det bara finns en person kvar.

### Filer som ingår i inlämningen

I ditt dotnet-projekt ska följande filer ingå
- `ICircularList.cs` med interfacet ICircularList enligt klassdiagrammet
- `CircularList.cs` med klassen CircularList enligt beskrivningen ovan
- `names.txt` med 100 namn som ska användas i det sista exemplet i `Program.cs`
- `Program.cs` där du illustrerar Josephus-problemet genom att använda din implementation av CircularList. Flera olika scenarion ska illustreras där du varierar antalet personer i leken, hur många steg som räknas framåt och om current flyttas till efterföljaren eller föregångaren när en person tas bort. Utskrifter ska göras så att det tydligt framgår i vilken ordning personer räknas ut och vem som är kvar.  
Slutligen ska du avsluta med ett exempel av Josephus-problemet där du har 100 personer i leken och räknar framåt 7 steg varje gång. De personer som ska placeras i den cirkulära listan finns på filen `names.txt` som du hittar i katalogen `example`.
- `AlternativRepresentation.txt` där du besvarar frågorna kring möjligheten att använda en array istället för en länkad struktur för att implementera en cirkulär lista.
