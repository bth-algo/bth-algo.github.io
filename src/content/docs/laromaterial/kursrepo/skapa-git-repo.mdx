Nu skall du skapa kursens Git-repo lokalt i din terminal under katalogen `dbwebb-kurser` genom att klona repot som du nyligen skapade på GitHub.

## Gå till katalogen dbwebb-kurser

Börja med att gå till din arbetskatalog för kurserna, den som du döpte till `dbwebb-kurser`.

```bash title="Gå till katalogen dbwebb-kurser"
# Gå till hemmakatalogen
cd

# Gå till dbwebb-kurser
cd dbwebb-kurser
```

Du kan göra samma sak i ett kommando och här refererar `~` till din hemmakatalog.

```bash title="Gå till katalogen dbwebb-kurser, i ett kommando"
cd ~/dbwebb-kurser
```

Använd kommandot `pwd` och `ls` för att försäkra dig om att du är på rätt plats i filsystemet.


## Klona ditt GitHub repo till ett lokalt repo

Leta upp ditt GitHub repo på [organistaionen bth-algo](https://github.com/bth-algo).

Leta reda på ett repo som heter algo-xxxx## där xxxx## är ditt studentkonto med fyra tecken och två siffror, tex abcd25 eller motsvarande.

Det är bara du själv och lärarna i kursen som har tillgång till repot.

Nu ska vi skapa en lokal kopia av ditt GitHub repo genom att klicka på "Code" och kopiera SSH-länken.

```bash title="Kommandon för att klona ditt kursrepo"
git clone git@github.com:bth-algo/algo-<akronym>.git
```

Om din akronym är `abcd25` så ser kommandot ut så här:  
`git clone git@github.com:bth-algo/algo-abcd25.git` 


### Kolla ditt lokala Git-repo

Du kan använda kommandot `pwd` och `ls` för att kontrollera vilken katalog du är i och vad den innehåller. Använder du `ls -la` får får du detaljer om filer och kataloger och det visas även dolda kataloger som börjar med en punkt `.`.

### Kör setup i ditt lokala Git-repo

Nu kan du köra följande kommando för att göra ditt kursrepo redo. Kommandot skapar kataloger, laddar ner workflow-fil & task-fil samt tester. Dessutom skapas första inlämningsbranchen `bth/submit/kmom03`.

```bash title="Kommando för att göra ditt lokala kursrepo redo"
pwd // visar att du står i ditt lokala repo dbwebb-kurser/algo-abcd25 för studenten abcd25

task setup
```

I slutet av setup-skriptet körs en del tester. Vid fel så kör du den kommandon som föreslås. Du kommer troligen att få köra kommandon för fixa till `bth/submit/kmom03`-branchen lokalt och nästa kommando är för att byta tillbaka till `main`-branchen.

När alla testerna gått igenom är det bara att köra på!

### Testa att ladda upp en lokal ändring till GitHub.

Flytta in i katalogen för ditt lokala Git-repo och gör en ändring i filen `README.md`. Gör sedan `git add`, `git commit` och `git push` till ditt GitHub-repo. `git commit -a -m "text` är samma sak som `git add .` följt av `git commit -m "text`.

```bash title="Flytta in till kursrepot"
# Flytta in i kursrepot, byt ut xxxx## mot din studentakronym
cd algo-xxxx##

# Titta vilka filer som finns i ditt lokala Git-repo 
ls -la
tree .
```

Öppna upp ditt lokala kursrepo i VS Code.

```bash title="Öppna repot i VS Code"
# Öppna nuvarande katalog i VS Code
code .
```

Gör en mindre ändring i filen `README.md` och pusha upp ändringen till GitHub för att verifiera att du har tillgång till repo.

```bash title="Gör commit på din ändring"
# Efter att du har redigerat din README.md
git status

git commit -a -m "Verify I have read/write access to the repo"
git push

git status
```

Dubbelkolla på GitHub att din push syns där!

### Om något går fel...

:::caution
Om något går fel så kan du radera din katalog (ditt repo lokalt) och försöka igen.

```bash title="Så här raderar du ditt lokala repo om du vill börja om från början."
# Change <dirname> to the name of your course repository name,
# for example algo-abcd25.
cd ~/dbwebb-kurser
rm -rf <dirname>
```

Kör därefter kommandona som klonar ditt lokala kursrepo igen och därefter setup-skriptet enligt ovan.
:::
