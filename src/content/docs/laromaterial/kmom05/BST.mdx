Ett binärt sökträd (eng Binary Search Tree), BST är ett binärt träd där:
- nodernas värden kan jämföras så att de kan ordnas sinsemellan
- för varje nod gäller att 
    - alla noder i dess vänstra delträd har lägre värden än noden
    - alla noder i dess högra delträd har högre värden än noden (om inga dubbletter tillåts, annars även samma värde)

Genom att använda binära sökträd kan vi "snabbt" hitta ett värde eftersom vi alltid kan bortse från det ena av de båda delträden till en nod.
Om trädet är relativt fullt på alla nivåer så blir sökningen O(log<sub>2</sub>n) precis som vid binärsökning (som ju bygger på att hälften av alla element "kastas" vid varje kontroll av mittelementet).

Nedan ser du 4 BST som innehåller samma värden. Kontrollera noga att träden uppfyller det som anges i punkterna ovan.

- sätta in ett element
- ta bort ett element

- bestämma minsta elementet
- få veta antalet element

## Insättning  

Vi vet att ett element som sätts in i ett BST ska följa regeln om mindre till vänster och större till höger. Vi utgår från roten och jämför med elementet för att se om det ska in i det vänstra ( < ) eller det högra delträdet. När vi har insett vilket delträd elementet ska placeras i flyttar vi till roten i detta delträd (som ju är barn till den nod vi precis utgick ifrån). Därefter fortsätter vi på samma sätt tills vi kommer till den nod där delträdet som elementet ska placeras i är tomt. Det nya elementet infogas här (som ett löv) och blir därmed delträd till noden som inte hade något barn.

En iterativ algoritm för insättning ges av:

```text
insert(element)
    if tree is empty
        insert element as root
    else
        currentNode = root
        inserted = false
        while not inserted
            if element < currentNode.Element
                if currentNode.LeftChild is none
                    add element in a node as leftChild
                    inserted = true
                else
                    currentNode = currentNode.LeftChild
            else
                if currentNode.RightChild is none
                    add element in a node as rightChild
                    inserted = true
                else
                    currentNode = currentNode.RightChild

```

Vi följer algoritmen för att sätta in följande element i ett från början tomt BST.  
55, 60, 12, 7, 89, 10

**insert(55)**: resulterar i att roten innehåller 55  

**insert(60)**: 60 > 55 (som är rotens element) vilket innebär att 60 ska placeras i höger delträd till 55. EFtersom 55 inte har något höger barn placeras 60 i en nod som blir höger barn till 55.

**insert(12)**: 12 < 55 (som är rotens element) vilket innebär att 12 ska placeras i vänster delträd till 55. EFtersom 55 inte har något vänster barn placeras 12 i en nod som blir vänster barn till 55.

**insert(7)**:  7 < 55 vilket innebär att 7 ska placeras i vänster delträd till 55. EFtersom 55 har ett vänster barn blir vänster barn, som innehåller 12, aktuell nod istället. Nu kontrolleras om 7 < 12, vilket är sant, och noden med 12 har inget vänster barn. En ny nod med värdet 7 tillförs som vänster barn till noden med 12.

**insert(89)**: 89 > 55, höger barn finns och innehåller 60. 89 > 60, höger barn finns inte och 89 blir därmed höger barn till 60.

**insert(10)**: 10 < 55, vänster barn finns och innehåller 12. 10 < 12, vänster barn finns och innehåller 7. 10 > 7, höger barn finns inte och 10 blir därmed höger barn till 7.

## Ta reda på om ett element finns

Vid sökning i ett BST tar vi nytta av att alla element i vänstra delsträdet är mindre och alla element i högra är större.   
Vi börjar i roten och undersöker om det element som söks finns i roten - i så fall är vi klara.   
Om så inte är fallet undersöker vi om det sökta elementet är mindre än det som finns i roten. I så fall ska vi fortsätta i det vänstra delträdet om det är möjligt, dvs om det finns ett vänsterbarn. Finns det inte ett vänster barn är vi klara och kan konstatera att det sökta elementet inte finns i trädet
Om det som söks istället är större än det som finns i roten fortsätter vi i det högra delträdet. Om det inte finns kan vi konstatera att elementet som söks inte finns i trädet.

Förfarandet är "snarlikt" hur insättningen genomförs.

1. Skriv en algoritm i pseudokod `find(elem)` för att bestämma om elem finns. Algoritmen ska returnera sant eller falskt.

    <details>
    <summary>Klicka här för att visa svaret</summary>  

    ```
    find(elem)
        currentNode = root
        while currentNode is not none AND currentNode.Element != elem
            if elem < currentNode.Element
                currentNode = currentNode.LeftChild
            else
                currentNode = currentNode.RightChild
        if currentNode is none
            return false
        return true

    ```

## Borttagning

Vid borttagning i ett BST identifieras 3 olika fall:

Det element som ska tas bort finns i:
1. ett löv
1. en nod som har 1 barn
1. en nod som har 2 barn

Borttagning av t.ex 4 eller 72 motsvarar fall 1, dvs att ta bort ett löv. Det är enkelt eftersom det inte påverkar noderna i övrigt i trädet. Lövet med elementet tas helt enkelt bort.

Borttagning av t.ex 40 eller 80 motsvarar fall 2, dvs att ta bort en nod med ett barn. Här kan den nod som tas bort ersättas med dess enda barn. Noden med 40 är höger barn till noden med 33 och om noden med 40 tas bort får noden med 58 (som är det enda barnet) bli höger barn till noden med 33.
På motsvarande sätt kommer noden med 74 att bli höger barn till noden med 65.

Borttagning av t.ex 33 eller 74 motsvarar fall 3, dvs att ta bort en nod som har två barn. Om vi utgår från 33 så vet vi att alla element i vänster delträd är mindre än 33 (och alla i höger är större). Om vi kan hitta det element i vänster delträd som är störst så skulle det kunna ersätta 33 (vi det ju att detta är större än alla andra i det vänstra delträdet och dessutom självklart mindre än alla i det högra delträdet eftersom alla element där är större än 33). Vi har sedan tidigare en algoritm för att hitta största (eller minsta) i ett BST. Eftersom vänster delträd självklart är ett BST kan algoritmen för att hitta det största lösa detta.  
Algoritmen ger oss 25. Nu kopierar vi elementet, dvs 25, till noden som innehåller 33 och därefter tar vi bort noden med 25 i det vänstra delträdet. Det resulterar i borttagning av en nod som har ett (1) barn (Bild 1) vilket vi har en lösning på sedan tidigare (Fall 2).

Om vi istället tar bort 74, kommer 72 (största elementet i vänster delträd) att kopieras och noden med 72 i den vänstra delträdet därefter att tas bort (Bild 2). Detta är ett löv, vilket vi också har en lösning på sedan tidigare (Fall 1).

Det första som ska ske vid borttagning är att hitta noden som innehåller det element som ska tas bort, vilket motsvarar `find(elem)` med justeringen att vi behöver noden som `elem` finns i och dessutom nodens förälder, eftersom dess barn kommer att tas bort (Fall 1 och Fall 2). Denna del kommer att vara inledande i borttagningsalgoritmen.

Vi kan dela upp detta i två delproblem

1. hitta noden för elementet `findNodeOf(elem)`
1. hitta föräldern till noden med elementet `findParentOf(theNode)`

Idé för `findNodeOf(elem)`:   
Utgå från roten genom att sätta en *pekare* till roten och uppdatera denna *pekare* med vänster eller höger barn (beroende på nodens värde jämfört med `elem`) så länge *pekaren* inte är "tom" och den nod den "pekar på" inte har ett innehåll som är detsamma som `elem`.

    ```
    findParentOf(theNode)
        currentNode = root
        parent = none
        while parent is none
            if currentNode.LeftChild == theNode or currentNode.RightChild == theNode
                parent = currentNode

            else if theNode.Element > currentNode.Element

    ```

Detta motsvarar en rekursiv definition och vi behöver därmed även basfall. Det får vi genom att inse att ett tomt träd innehåller 0 noder. Vi får:

```
nrOfNodes(currentRoot) = nrNodes(currentNode.LeftChild) + nrOfNodes(currentNode.RightChild) + 1  
nrOfNodes(none) = 0
```

## Representera BST

Vid implementation av BST används länkning med en klass för att representera Node. Klassen Node behöver pekare (referensvariabler) för att peka på vänster barn och för att peka på höger barn.   
Eftersom elementen som ska placeras i trädet ska ordnas krävs det att deras datatyp är jämförbar. I C# betyder detta att interfacet `IComparable<T>` används. Klassen Node placeras lämpligen som inre klass inom klassen som representerar BST.   

Det finns en möjlighet att visualisera trädet genom att skapa en sträng med en viss uppbyggnad. Denna del är klar genom metoden `toGraphviz()` som med hjälp av `BuildGraphviz()` konstruerar och returnerar en sträng som har en uppbyggnad som visualiseringsverktyget använder. Visualiseringsverktyget hittar du här [GraphvizOnline](https://dreampuf.github.io/GraphvizOnline/). 

Här är ett exempel på hur GraphvizOnline används:

```csharp
BST<int> bst = new BST<int>();
bst.Insert(44);
bst.Insert(77);
bst.Insert(11);

Console.Write(bst.toGraphviz());
```

Detta resulterar i utskriften:

```text
digraph BST {
    node [shape=circle];
    "44" -> "11"[color=blue];
    "44" -> "77"[color=red];
}
```

Denna utskrift kopieras och klistras in i den vänstra delen och genast visas trädet i den högra delen.

## Sammanfattningsvis

Ett BST (Binärt Sökträd) är ett binärt träd med regeln att för varje nod gäller att alla värden i vänster delträd är mindre än nodens värde och alla värden i höger delträd är större än nodens värde.  

Vid insättning hamnar det nya värdet i ett löv eftersom man bestämmer platsen genom att vandra från roten och jämför om det nya värdet ska till vänster eller höger tills dess att en ledig plats hittas (vilket alltid blir ett löv).  

Sökning utförs på motsvarande sätt avseende vänster och höger delträd.

Vid borttagning finns tre olika fall som behöver hanteras:  
1. att ta bort ett löv
1. ta bort en nod som har ett barn
1. ta bort en nod som har två barn
